## Приложение: шпаргалка по Git Flow

1. `git checkout main` → `git pull` → `git checkout -b develop` → `git push -u origin develop`.
2. Для каждой фичи: от `develop` создать `feature/имя`, коммиты, слить в `develop`, `git push origin develop`.
3. Когда готово: `git checkout main` → `git merge develop` → `git push origin main`.

Подробные команды см. в методичке или в истории переписки; зафиксируйте в отчёте фактические имена веток и коммитов.
